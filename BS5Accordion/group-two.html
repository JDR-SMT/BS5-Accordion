Programming controls dictate the execution flow within a program. The three primary flow control types include sequence, selection and iteration. Programming design structures promote readable code and reusable components when developing programs to minimise code redundancy and errors.

Website testing procedures include: functionality testing, ensures all components of the website are working correctly; usability testing, determines if the user interface of the website is intuitive and user-friendly; compatibility testing, ensures the website functions correctly across various device, operating system and browser combinations; cross-browser testing, ensures the website functions and displays correctly across various browsers; responsive testing, ensures the layout and design of the website maintains consistency when adapting to various devices and screen sizes; accessibility testing, determines adherence to WCAG accessibility standards; performance testing, determines the responsiveness of the website; user acceptance testing (UAT), gathers feedback from users to evaluate whether user requirements and expectations are met; regression testing, involves thoroughly testing updates prior to being released; and localisation testing, if applicable, ensures the website functions and displays correctly in all languages.

Website debugging methods include: black box testing, methods are hidden to focus on ensuring inputs and outputs meet requirements; commenting out, enables line by line error identification; stress testing, evaluates the reliability and stability of the website during peak traffic load; breakpoints, evaluates website consistency when adapting to various devices and screen sizes by setting breakpoints at media queries; print statements, displays the output of identified errors; report and error tracking, ensures errors are reported, prioritised and monitored; and DevTools, inbuilt browser DevTools provides a range of tools to diagnose errors.

Website coding techniques include: ensure thorough planning; utilise coding standards; ensure code is written with purpose; utilise comments; develop and update incrementally; ensure device and browser compatibility; and implement a contact page.
